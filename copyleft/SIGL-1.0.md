A copyleft license requiring cryptographic signing of all distributed source code.

## Preamble

The SIGL adds a security layer to copyleft: all distributed source code and binaries must be cryptographically signed by a verified developer key. This ensures source code authenticity and prevents supply chain attacks where modified code is distributed under the original author's name.

## TERMS AND CONDITIONS

### 0. Definitions.

"Developer Signature" means a cryptographic signature using a private key associated with a verified developer identity.

"Signed Release" means a release where the source code archive and any binary artifacts are accompanied by a Developer Signature.

### 2. Signature Requirement.
