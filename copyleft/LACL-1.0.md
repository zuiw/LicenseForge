A copyleft license with tiered requirements based on software layering.

## Preamble

The LACL recognizes that software operates in layers (kernel, core library, plugin, application), and applies different copyleft strength to each. The core layer carries the strongest obligations, while plugins, extensions, and higher-level applications have increasingly lighter requirements.

## TERMS AND CONDITIONS

### 0. Definitions.

"Core Layer" means the primary functional code of the Licensed Work.

"Extension Layer" means code that extends or adds features to the Core Layer through documented APIs.

"Application Layer" means end-user applications that use the Core or Extension Layers.

### 1. Permissions.

You may use, copy, modify, and distribute the Licensed Work, subject to the layer conditions below.

### 2. Core Layer (Strong Copyleft).

Modifications to the Core Layer must be licensed under LACL-1.0 with full source disclosure.

### 3. Extension Layer (Weak Copyleft).

Modifications to the Extension Layer must be made available as source code to users but may use a compatible license.

### 4. Application Layer (No Copyleft).

Applications using the Licensed Work through documented interfaces need not disclose source code and may use any license.

### 5. Layer Boundary.

The Licensor must document which portions of the codebase constitute each Layer.
