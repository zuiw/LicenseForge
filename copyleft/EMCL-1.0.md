## Preamble

The EMCL is tailored for embedded software, firmware, and IoT devices. It requires source disclosure for the Licensed Work and its modifications, but provides exceptions for bootloaders, hardware initialization code, and third-party closed firmware running on separate processors.

## TERMS AND CONDITIONS

### 0. Definitions.

"Embedded System" means a dedicated computer system with limited resources performing specific functions.

"Firmware" means software stored in non-volatile memory controlling hardware.

### 1. Permissions.

You may use, copy, modify, and distribute the Licensed Work, subject to the embedded conditions below.

### 2. Embedded Copyleft.

Modifications to the Licensed Work in an Embedded System must be disclosed under EMCL-1.0.

### 3. Bootloader Exception.

Bootloader code and hardware initialization routines are not subject to disclosure requirements.

### 4. Co-Processor Exception.

Software running on separate processors (e.g., baseband, Wi-Fi module) is not subject to copyleft.

### 5. Termination.

Distributing modified Embedded System firmware without disclosure terminates rights. A 90-day cure period applies.
