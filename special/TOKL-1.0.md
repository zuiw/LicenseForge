## Preamble

The TOKL governs smart contracts and software implementing token systems. It requires full disclosure of tokenomics, including supply schedules, vesting terms, inflationary/deflationary mechanisms, and any administrative control functions. It ensures that token users can make informed decisions.

### 2. Supply Transparency.

The Software must disclose:
- **a)** Total Supply and emission schedule;

### 3. Vesting Disclosure.

If Tokens are allocated to founders, investors, or team members, the vesting schedule must be publicly documented, including cliff periods, release schedules, and unlock conditions.

- **a)** Document what each key can do;
